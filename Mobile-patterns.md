# Мобильный дизайн. Христоматийный

## Краткая вводная по сегодняшней лекции
Компоненты нужны для создания интуитивно понятного интерфейса. Используя знакомые элементы (например, стандартную кнопку или поле ввода), дизайнер обеспечивает мгновенное узнавание: пользователь сразу понимает, как с ними взаимодействовать. Повторное использование готовых компонентов повышает последовательность дизайна – одинаковые элементы ведут себя одинаково во всех частях приложения, не сбивая пользователя с толку. Кроме того, опора на стандартные компоненты облегчает разработку: многие из них реализованы в платформах Android/iOS и хорошо задокументированы​
. Таким образом, следование гайдлайнам платформ (стандартам дизайна) и применение нативных элементов ускоряет работу и снижает количество ошибок при реализации UI​
. В итоге выигрывают все: дизайнер быстрее создает понятный интерфейс, разработчик – использует готовые решения, а пользователь – получает привычный опыт.

## Какие компоненты мы сегодня изучили?


### Базовые компоненты мобильного дизайна — краткое резюме

#### Навигация
**iOS**:
- *Tab Bar* — основной способ навигации, всегда снизу. Поддерживает 2–5 разделов + «Ещё».
- Не используется «бургер-меню».

**Android**:
- *Bottom Navigation Bar* — аналог Tab Bar, активно используется в новых приложениях.
- *Navigation Drawer* (бургер-меню) допустим при большом числе разделов.

**Особенности**:
- iOS делает ставку на постоянную доступность вкладок.
- Android допускает больше вариативности (в т.ч. Drawer).

---

#### Tabs (вкладки внутри экрана)

**Android**:
- Tabs с возможностью свайпа между экранами.
- Не рекомендуется добавлять горизонтальные карусели внутри.

**iOS**:
- *Segmented Control* — переключатель контента без свайпа.
- Используется чаще для фильтров, чем навигации.

**Вывод**: не путать Tabs и Tab Bar. Таблицы Android можно адаптировать на iOS через другие элементы (например, Page Control).

---

#### Верхняя навигация (App Bar / Navigation Bar)

**Android**:
- *App Bar* (Toolbar): заголовок, иконки, меню, кнопка «Назад» или бургер.
- Есть системная кнопка/жест назад.

**iOS**:
- *Navigation Bar*: заголовок + кнопка Back, жест возврата свайпом слева.
- Back **обязателен** в UI, если нет жеста.

---

#### Кнопки

**FAB (Floating Action Button)**:
- Только на Android.
- Главная кнопка действия, «плавает» над экраном.

**Обычные кнопки**:
- Android: Text, Contained, Outlined, Toggle.
- iOS: текстовые, с заливкой, обычно скруглённые.

**Обратная связь**:
- Android: ripple-эффект.
- iOS: затемнение, прозрачность.

---

#### Кнопки с иконками

- Присутствуют на обеих платформах.
- Используются в App Bar / Navigation Bar.
- Размеры: минимальная зона нажатия 44pt/48dp.
- Использовать стандартные библиотеки иконок: **Material Icons**, **SF Symbols**.

---

#### Формы и поля ввода

**Android**:
- Floating Label, рамка, стили Filled/Outlined.
- Чёткие состояния фокуса, ошибки, цвета.

**iOS**:
- Placeholder без рамки.
- Метка чаще отдельная, минимализм.
- Менее строгое оформление по HIG.

---

#### Прочие элементы форм

**Переключатели**:
- Android: Checkbox (множ. выбор), Radio (одиночный), Switch.
- iOS: только Switch и списки с галочками.

**Списки выбора**:
- Android: Dropdown / Spinner.
- iOS: Picker (колесо), Action Sheet.

**Вывод**: используйте родные контролы платформы. Не переносите checkbox в iOS — адаптируйте под стандартные списки.

---

## Таблица сравнения компонентов **Android vs iOS**
|**Компонент / паттерн**|**Android (Material Design)**|**iOS (Human Interface Guidelines)**|
|---|---|---|
|**Основная навигация**|Bottom Navigation Bar (нижняя панель 2–5 разделов) или Navigation Drawer (бургер-меню для 5+ разделов). Верхний App Bar с заголовком и меню.|Tab Bar (нижняя панель вкладок 2–5 разделов, с кнопкой “Ещё” для дополнительных). Верхний Navigation Bar с заголовком и кнопкой Back.|
|**Кнопка возврата (Back)**|Системная кнопка/жест Назад (Android Navigation Bar) + иконка-стрелка в App Bar при необходимости.|Кнопка “Back” в левом верхнем углу Navigation Bar или свайп от края экрана для возврата​.|
|**Кнопки действий**|Floating Action Button (круглая, акцентная) для основного действия. Обычные кнопки: Contained (с заливкой), Text, Outlined. Иконки действий в App Bar (поиск, меню).|FAB отсутствует. Основные действия – кнопки на экране (например, большая кнопка внизу) или в Navbar (правый верхний угол). Стиль кнопок – текстовые (синие по умолчанию) или с легким фоном, скругленные.|
|**Поле ввода (Text Field)**|Материальное текстовое поле с плавающим лейблом, стилем Filled/Outlined, выделением цветом при фокусе. Варианты: одиночная строка, многострочное, с иконкой (поиска) и т.д. Предусмотрены подсказки и сообщения об ошибке.|Стандартное поле UITextField с placeholder (текстом-подсказкой внутри). Нет плавающей метки – обычно ярлык поля пишется отдельно сверху. Стиль – подчеркивание или рамка вокруг. Обработка ошибок оставлена на усмотрение (может быть текст ошибки под полем, но HIG не регламентирует строго).|
|**Переключатели**|Checkbox (флажок) для множественного выбора, Radio Button для исключающего выбора, Toggle Switch для включения/выключения настроек.|Чекбоксы/радио не используются в UI – вместо них списки с галочками или сегментированный контроль. Переключатель UISwitch для вкл/выкл, выглядит как зелёный тумблер.|
|**Списки**|ScrollView/RecyclerView со списком карточек или строк. List Item с ripple-эффектом при нажатии. Можно группировать списки в категории с заголовками. Разделители (линии) не обязательны, часто заменяются отступами.|UITableView (Plain или Grouped) – списки с автоматическими разделителями между ячейками. При нажатии ячейка подсвечивается серым. Для иерархий используется Disclosure Indicator (стрелка) справа, показывающая, что можно нажать и перейти. Группированные списки имеют вид секций на сером фоне с округлыми уголками.|
|**Карточки (Cards)**|CardView/Material Card: прямоугольный блок с тенью (Elevation), отступами. Используется для контента в списках, превью и т.п. Карточки могут содержать изображение, текст, кнопки.|Явно “Card” не предусмотрен, но используются схожие блоки с закругленными углами и без тени (например, фоновые панели). Часто контент располагается просто на белом фоне без рамок, а для выделения секций применяют размытые панели или доп. отступы.|
|**Диалоги и алерты**|AlertDialog: всплывающее окно с текстом и кнопками (“OK/Cancel”). Стиль Material: прямоугольник с темой приложения, кнопки обычно внизу справа. Snackbar: ненавязчивое сообщение внизу с опцией действия (например, “UNDO”).​|UIAlertController (Alert): модальное окно с заголовком и списком кнопок (Cancel, другие опции). Стиль iOS: округлые края, размытый фон. Action Sheet: список вариантов, выезжающий снизу (для ряда действий). Краткие уведомления: системные баннеры вверху экрана (если необходимо показать сообщение и не ждать действия).|
|**Индикаторы загрузки**|Circular Progress Indicator (спиннер) – обычно материал-дизайн кольцо, может быть цветным. Linear Progress – полоска прогресса (цветная). Также могут быть кастомные индикаторы (например, индeterminate “шапка” вокруг кнопки). Refresh Indicator – при pull-to-refresh показывается крутящееся кольцо Material.|UIActivityIndicator (спиннер) – стандартный серый/белый индикатор, малый размер. UIProgressView – синий прогресс-бар. Pull-to-refresh – стандартный индикатор в UITableView (спиннер около верхнего края списка). Системный индикатор сети убран из статусбара в новых версиях, поэтому явные индикаторы в интерфейсе используются чаще.|
